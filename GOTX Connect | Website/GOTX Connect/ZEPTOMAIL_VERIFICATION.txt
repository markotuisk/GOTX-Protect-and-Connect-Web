ZEPTOMAIL VERIFICATION - GOTX CONNECT

QUESTION 1:
Please share the name and screenshot of the application using which you will be sending the transactional emails.

ANSWER:
Name: GOTX Connect. Website (Custom Application). www.gotx.uk

Description:
Our application is the official company website (www.gotx.uk). We use the ZeptoMail API to send the following transactional emails:
1. Automated inquiry receipts to customers who submit our "task brief" contact form.
2. Internal alerts to our staff for new task inquiries.
3. Welcome emails to users who voluntarily opt-in to our "Stay Involved" mission list.

All emails are strictly transactional and triggered exclusively by specific user actions on the website.

--

QUESTION 2:
Please provide the backend details where you will be adding the SMTP/API information of ZeptoMail.

ANSWER:
Infrastructure: 
Our application is hosted on Cloudflare Pages. The backend logic is handled via Cloudflare Pages Functions (a secure, serverless environment).

Integration:
We have integrated the ZeptoMail REST API into our server-side JavaScript functions (located at /functions/api/submit.js and /functions/api/subscribe.js). These functions execute only on the server ('The Edge'), ensuring the API credentials are never exposed to the client-side browser.

Security:
The ZeptoMail API Token is stored securely as an Environment Variable (ZEPTOMAIL_TOKEN) within the encrypted Cloudflare settings dashboard. It is not hardcoded in the source files and is physically inaccessible to anyone visiting the website.
